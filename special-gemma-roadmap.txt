# ModelWeaver Roadmap Refinement (Gemma-4 Special Edition)

This document serves as a detailed proposal for the evolution of ModelWeaver from V0.6 to V1.0. It expands on the user's vision with technical specifications and strategic additions.

---

## 🚀 V0.6 : Key Manager & LLM Abstraction
**Focus**: Security, autonomy, and unified access to LLMs.

### 1. Key Manager (Security & GUI)
- **Vault Implementation**:
    - Encryption: Use AES-256 (via `cryptography` lib) to encrypt API keys before storing them in SQLite.
    - Master Password: Implement a master password system to derive the encryption key (PBKDF2).
- **GUI Vault**:
    - Interface to add/edit/delete keys.
    - "Health Check" button: Verify each key's validity via a simple API call.
    - Auto-Discovery: Suggest providers based on the key format (e.g., `sk-` for OpenAI/Groq).

### 2. LLM Wrappers & Bridge
- **Unified Wrapper**: Create a high-level abstraction layer so agents don't care if they use Ollama, LiteLLM, or a direct API.
- **Ollama Wrapper**: Dedicated manager for model pulling, listing, and health monitoring.
- **Universal Bridge**: Ensure that any new LLM provider can be added by simply creating a new wrapper class.

---

## 🛠️ V0.7 : Agent & Role Studio
**Focus**: Low-code agent creation and operational validation.

### 1. Role GUI (The "Blueprint" Editor)
- **Visual Prompting**: Editor for system prompts with variables (e.g., `{{project_context}}`).
- **Config Panel**: Sliders for temperature, top_p, and max_tokens.
- **Tool Mapping**: Check-box list of available tools from the Catalogue that this role can use.

### 2. Agent Deployment GUI
- **Instancing**: Create an agent instance by linking (Role $\to$ Model/Provider $\to$ Constraints).
- **Instance Management**: Ability to run multiple agents of the same role with different configurations.

### 3. Functional Validation
- **Sandbox Test**: A "Run Test" button that triggers a specific prompt and checks if the agent responds and uses tools correctly.
- **Log Trace**: Real-time view of the agent's thought process during the test.

---

## 📊 V0.8 : Control Center & Visual Orchestration
**Focus**: Monitoring and high-level pipeline design.

### 1. The Dashboard (Tauri GUI)
- **Agent Grid**: Visual status of all agents (🔴 Stopped, 🟡 Idle, 🟢 Working).
- **Live Stream**: Unified log viewer with filters by agent or session.
- **Resource Monitor**: CPU/RAM usage per agent (integrating `psutil`).

### 2. Visual Graph Orchestration
- **Node-Based Linking**: A drag-and-drop interface to connect agents.
- **Pipeline Logic**: 
    - Define sequences (Agent A $\to$ Agent B).
    - Define loops (Agent B $\to$ Agent C $\to$ Agent B if not validated).
    - Define branches (If result = X $\to$ Agent D).
- **Orchestration Sync**: Automatically update the `AgentQueue` and `WatcherService` based on the visual graph.

---

## 🧪 V0.9 : Efficiency Benchmark (Local CrewAI)
**Focus**: Proving the power of the system with 0$ budget.

### 1. The "Zero-Cost" Challenge
- **Scenario**: Coding a full-stack project (e.g., a Task Manager with FastAPI + React) from scratch.
- **Stack**: Use exclusively local models (Ollama) or free tiers (Groq, Gemini).
- **Crew Setup**: 
    - 1 Architect $\to$ 1 Coder $\to$ 1 Tester $\to$ 1 Debugger.
- **Metrics for Success**:
    - **Autonomy**: Number of human interventions required.
    - **Quality**: Percentage of passing tests on first run.
    - **Coherence**: Consistency of the codebase across different agents.

---

## 🪟 V0.10 : Windows Adaptation
**Focus**: True cross-platform availability.

### 1. OS Abstraction Layer
- **Shell Commands**: Replace bash-specific scripts with Python-native calls or PowerShell equivalents.
- **Pathing**: Full audit of all `Path` usages to ensure no hardcoded `/` or `C:\`.
- **Binary Management**: Implementation of Windows-specific installers (winget/choco).

### 2. Windows-Specific Build
- **Tauri Windows**: Optimize the GUI for Windows 10/11.
- **DLLs & Dependencies**: Ensure all C-extensions (like `libsql`) are correctly bundled for Windows.

---

## 📦 V0.11 : Public Readiness
**Focus**: Polishing, stability, and transparency.

### 1. Code Hardening
- **Global Error Handling**: Implementation of a centralized exception manager to prevent crashes.
- **Logging**: Shift from `print` to a structured `logging` system (files + console).
- **Secrets Audit**: Automated scan to ensure no internal keys are shipped.

### 2. Documentation
- **Contributor Guide**: How to add new recipes or modules.
- **API Reference**: Documentation for the `Plumber` and `AgentFactory`.

---

## 💻 V0.12 : Field Testing & Debugging
**Focus**: Real-world validation on diverse hardware.

### 1. Multi-Machine Beta
- **Hardware Diversity**: Testing on low-end laptops (8GB RAM) and high-end workstations.
- **OS Mix**: Testing on various Ubuntu flavors and Windows versions.
- **Network Stress**: Testing with slow connections (remote recipes) and offline modes.

---

## 🎨 V0.13 : Branding & Aesthetic Overhaul
**Focus**: User experience and identity.

### 1. The "New Face"
- **Rebranding**: Define a new name, logo, and color palette.
- **UX Refactor**: Simplify the GUI flow to make it "One-Click" friendly.
- **CLI Polish**: Implementation of `Rich` for beautiful terminal outputs (tables, panels, progress bars).

---

## ⚖️ V0.14 : Finalization & Strategy
**Focus**: Legal, structural, and marketing foundations.

### 1. Legal & Manifest
- **License**: Define a clear license (e.g., AGPL for the core, proprietary for some plugins).
- **Manifest**: A comprehensive list of all supported tools, models, and their capabilities.
- **README 2.0**: Professional landing page with GIFs, architecture diagrams, and a quick-start guide.

### 2. Marketing Strategy
- **Value Proposition**: "The only local-first AI orchestrator that installs its own tools."
- **Channels**: Plan for Reddit (r/LocalLLM), HackerNews, and YouTube.

---

## 🚀 V1.0 : The Big Bang
**Focus**: Launch and growth.

- **Official Release**: GitHub release with pre-compiled binaries.
- **Community Growth**: Open an Issue tracker for feature requests and bug reports.
- **Promotional Campaign**: Launch on social media and AI forums.
